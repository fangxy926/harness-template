# 登录页面设计文档

## 概述

为 Web 前端单页应用提供一个自包含的静态登录页面，使用纯 HTML + CSS + JS 实现，零依赖，无需构建工具。

## 目标

- 提供用户名 + 密码登录表单
- 前端表单验证（非空校验、密码最小长度）
- 密码可视性切换
- 模拟登录成功反馈（无真实后端）

## 非目标

- 不接入任何后端 API
- 不涉及 Session / Token 管理
- 不支持第三方 OAuth 登录
- 不包含注册、忘记密码页面

---

## 文件结构

```
login.html    # 唯一产物，CSS 内联在 <style>，JS 内联在 <script>
```

单文件设计，直接用浏览器打开即可运行。后续可将 CSS/JS 提取为独立文件，HTML 结构保持不变。

---

## UI 组件

### 页面布局

```
<body>                          ← 全屏背景（深色渐变）
  <div class="card">           ← 居中白色卡片（max-width: 400px）
    <h1>登录</h1>
    <form id="login-form">
      <label + input[text]>    ← 用户名输入框
      <label + input[password] + 👁按钮>  ← 密码输入框
      <button[submit]>         ← 登录按钮（全宽）
    </form>
    <p class="error-msg">      ← 错误提示（默认 hidden）
  </div>
</body>
```

### 视觉规格

| 属性 | 值 |
|------|-----|
| 背景渐变 | `#1a1a2e → #16213e`（上→下） |
| 卡片背景 | `#ffffff` |
| 卡片圆角 | `12px` |
| 卡片阴影 | `0 20px 60px rgba(0,0,0,0.3)` |
| 主色调 | `#4F46E5`（靛蓝） |
| 字体 | 系统默认 sans-serif |
| 错误文字色 | `#ef4444` |

## 交互行为

### 表单验证（提交时触发）

| 场景 | 错误提示文案 |
|------|------------|
| 用户名为空 | 请输入用户名 |
| 密码为空 | 请输入密码 |
| 密码长度 < 6 | 密码至少 6 位 |

- 输入框获得焦点（focus）时，自动清除对应错误提示

### 登录成功模拟

1. 校验通过后，按钮文字变为「登录中…」并禁用（持续 500ms）
2. 500ms 后，显示「登录成功 ✓」绿色提示
3. 不发送任何网络请求

### 密码可视性切换

- 点击密码框右侧 👁 图标，切换 `input[type]` 在 `password` / `text` 之间
- 图标随状态变化（👁 / 🙈）

---

## 错误处理

- 所有错误仅通过页面内 `.error-msg` 元素展示，不使用 `alert()`
- JS 使用 `try/catch` 包裹提交逻辑，防止意外异常静默失败

---

## 验收标准

1. 单独打开 `login.html` 无控制台报错
2. 空表单提交，正确显示对应错误提示
3. 密码 < 6 位时，显示长度校验错误
4. 校验通过后，出现 500ms loading → 登录成功提示
5. 👁 按钮可切换密码显示/隐藏
6. 页面在 Chrome / Firefox / Edge 最新版正常渲染
